## ✨ 主要变化

### 1. Skill 动态注入（最大 token 节省）

**之前**：`skill:loadDefault` 一次性拼 7 个 Skill 模板到 system prompt（~4K tokens），Agent 永远看到全部 Skill。

**现在**：Agent 可传 `skills=['ingest', 'query']` 只拼触发的 1-2 个 Skill 详情，**省 ~3-4K tokens / 每次启动**。

```typescript
// Agent 启动
const skillPrompt = await window.api.skillLoadDefault(['ingest', 'query'])
// 只返 ingest + query 详情 + 始终含 caps (编辑器能力)
// 省 5 个无关 Skill 的 token
```

**`composeInjectedSkillText(vaultPath, skills?)` 行为**：
- `undefined`/空数组 → 拼全部 7 个（v1.5 行为，向后兼容）
- `['ingest']` → 只拼 ingest
- `['ingest', 'lint']` → 拼 ingest + lint（按文件名字母序）
- 不存在的 skill 名 → 静默忽略
- `MARKDOWN_CAPABILITIES.md` 始终拼（编辑器能力轻量）

### 2. query:vault 过滤（搜索效率）

**之前**：`query:vault(question)` 固定 50 LIMIT，不传过滤，Agent 拿到 50 条 source 喂 AI。

**现在**：
```typescript
const result = await window.api.queryVault('ABC 合同', {
  topic: '合同管理',          // 只看 _wiki/合同管理/ 目录
  maxResults: 10,             // 截断 FTS5 结果到 10
  maxWikiFiles: 8             // 每个 topic 目录遍历 8 个文件（默认 6）
})
```

**省 ~50% 搜索 token**（50 source → 10 source + Step 0 跳其他 topic）。

### 3. kg:queryTopics 新增 IPC（KG 文本查询）

**之前**：Agent 看 KG 只能视觉，**不能**文本查节点——只能让用户截图描述，不准 + 费 token。

**现在**：
```typescript
// 查 "ABC 合同" 相关的 KG 节点
const result = await window.api.graph.queryTopics('ABC', {
  maxNeighbors: 10,    // 每匹中节点最多 10 条边
  maxResults: 50       // 匹中节点数上限
})
// → { query: 'ABC', nodes: [...], edges: [...] }
```

**实现**：纯 in-memory 过滤 `loadGraph()` 结果，**不改 graph service**。name 不传 = 全部节点 + 边（LIMIT 500 防暴）。

### 4. briefing:getConversations 过滤（记忆读取效率）

**之前**：`briefing:getConversations(date)` 返当日所有摘要，vault 用久了可能 20-30 条。

**现在**：
```typescript
const summaries = await window.api.getConversations('2026-06-05', {
  topic: '合同管理',         // 只返该 topic 的摘要
  maxResults: 10             // 截断条数
})
```

**省 ~60% 记忆读取 token**。跨日期查同 topic 历史决策尤其省。

## 🔧 Agent 集成示例

v1.7 接口设计原则：**所有新参数都可选，向后兼容**。Agent 可以渐进式采用。

```typescript
// 之前 (v1.6.x) — 全部默认行为
await window.api.skillLoadDefault()
await window.api.queryVault(question)
await window.api.getConversations(date)

// 现在 (v1.7.0) — 优化调用
await window.api.skillLoadDefault(['ingest'])           // 只 1 个 skill
await window.api.queryVault(question, { topic, maxResults: 10 })
await window.api.getConversations(date, { topic, maxResults: 10 })
await window.api.graph.queryTopics('ABC')                // 新 IPC
```

## ⚠️ 内部教训

- v1.7 写 `briefing:getConversations` 时，**改了 IPC + 函数签名接 `options` 但漏实现过滤代码**。`@typescript-eslint/no-unused-vars` 报 `'options' is defined but never used`——**Lint 报 unused args 不算"功能完整"**。本版本 v1.7.0 commit `f34f1c7` 是补这次漏实现。
