## 🚨 必看：v1.6.1 / v1.6.2 dmg 启动闪退

**如果你在 macOS 26 + Apple Silicon（M1/M2/M3/M4）上跑 v1.6.1 / v1.6.2 dmg 启动崩溃**——本版本修复。

崩溃信息：EXC_BREAKPOINT (SIGTRAP) 在 `ares_llist_replace_destructor` + c-ares DNS 解析链。

**根因**：Squirrel.framework（Electron 模板自带的 Mac 自动更新器）在 macOS 26 + Apple Silicon 启动时崩 c-ares 链路。Squirrel 1.0 是 32-bit Mach-O 时代设计，现代 macOS 26 已不兼容。Free 仓库**不用 autoUpdater**，删后启动正常。

**升级**：
1. 下载 v1.6.3 dmg（见上表）
2. 拖到 /Applications 替换
3. 启动 — **应该不闪退**了

## ✨ 主要变化

### 1. dmg 启动闪退修复（最关键）

- **删 Squirrel.framework**：`scripts/after-pack.js` 新建（electron-builder afterPack hook 在 dmg pack 阶段删 Squirrel），`electron-builder.json` 加 `afterPack: scripts/after-pack.js`
- **修历史配置 bug**：`resources/whisper` extraResources 在 v1.6.2 删了文件但配置没删，造成 build 警告 `file source doesn't exist`
- **新 dmg 不再带 Squirrel.framework**：`find dist -name 'Squirrel*'` 匹不到
- **Frameworks 剩**：Electron Framework + Mantle + ReactiveObjC + 4 Helper.app（全 Electron 必要，不能删）

### 2. UI 设计契约一致性（对照 Pro 仓库 `docs/UI.md` v2.0）

按 UI.md 设计契约逐条核对 + 修复：

| 契约 | v1.6.2 状态 | v1.6.3 状态 |
|------|-------------|-------------|
| #1 颜色用 `var(--color-*)` | 656 处，0 硬编码 | 656 处，0 硬编码（不变）|
| #2 间距 4px 网格 `var(--space-*)` | 70+ 处硬编码 | **全清**（21 文件 187 insertions）|
| #3 动效 `var(--transition-*)` | 28 处 | 28 处 + **面板 transition 统一**（3 面板）|

**具体修改**：
- **focus-visible 全局 fallback**（`global.css` 末尾）：`* { outline: 2px solid var(--color-primary) }` + `input/textarea/contenteditable` 排除
- **4px 网格间距全清**（21 文件）：`6px → var(--space-2)` / `10px → var(--space-2)` / `12px → var(--space-3)` 等
- **面板 transition 统一**：LintPanel 健康检查/刷新按钮 + MemoryPanel 刷新按钮 + BacklinksPanel hover，加 `transition: var(--transition-base)`
- **抽 ThemeToggleButton 小组件**：3 处共享 inline style 复用

### 3. CSS gap 失效真因修复

v1.6.3 commit `1d2f1f8` 修缺逗号时**同时产生新 bug**：sed 替换 `gap: 4,fontSize: 12` 时把 `4` 替换成 `'var(--space-1)'`，原始 `,` 残留成前导逗号 → `gap: ',var(--space-1)'`（字符串值是 `,var(--space-1)`）。React 渲染时 CSS 收到 `gap: ,var(--space-1)`（前导逗号）—— **CSS 无效，gap 完全不生效**。

**30 处组件 UI 间距全错**（OutputPanel / LogPanel / WelcomeScreen / SchemaPanel / Editor / MemoryPanel / SearchPanel / TrashPanel / FloatingPanel / VaultCreationWizard / VersionHistoryPanel / LintPanel / SettingsSections）

**测试 196/196 + Lint 0 errors 都过了**（没真测 CSS 渲染）。v1.6.3 commit `9373099` 用直接字符串替换 `gap: ',var(--space-` → `gap: 'var(--space-`，30 处一次性清。

### 4. pre-commit hook 防御自动化

防御 v1.5 commit `b453511` 那种"批量加 memo 漏 import" bug 再犯（v1.5 漏 10 个文件，dev 跑 ReferenceError）：

- **`.githooks/pre-commit`**：跑 `check-memo-import.sh` + `npm run lint`
- **`scripts/install-hooks.sh`**：装到 `.git/hooks/`
- **`package.json`** 加 `precommit:install` 脚本
- 用户首次使用：`npm run precommit:install`
- 改 .tsx 提交时自动跑 hook，漏 memo import 时 commit 被拒绝
- **不依赖 husky**（git 社区标准，跨项目可复用）
