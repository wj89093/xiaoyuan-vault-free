## 4. 风险与备选方案

### 风险
- **FileTreeNode 重构可能 break 现有交互**(拖拽/键盘/hover)
- **FixedSizeList 性能 overhead** 在小 vault(20 文件)可能更慢
- **react-window API 限制**(itemData 引用更新需要 keyed forceUpdate)

### 备选
- 如果重构成本太大,考虑用 `react-virtuoso`(API 更宽容)
- 或只虚拟化深层(深度 ≥ 3)子项,顶层保留原生渲染

### 决策
先尝试 react-window 方案,跑不通再切。
