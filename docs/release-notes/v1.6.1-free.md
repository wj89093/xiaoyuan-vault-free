# v1.6.1-free Release Notes

> 发布日期：2026-06-05
> 状态：GA (General Availability)

## 一句话

晓园 Vault v1.6.1-free 是一次**质量收尾**版本——核心特性 v1.6.0 已就位（9 个独立 Skill 模板、Skill 模板 ↔ UI 接口对齐），本版本专注于**修对齐、修 bug、清 Pro 残留**。

## 📦 下载

| 平台 | 文件 | 大小 |
|------|------|------|
| macOS (Apple Silicon) | `晓园-Vault-1.6.1-free-arm64.dmg` | **~146 MB** (-29%) |
| macOS (Intel) | `晓园-Vault-1.6.1-free.dmg` | **~151 MB** (-29%) |
| Windows | `晓园-Vault-Setup-1.6.1-free.exe` | （CI 出）|
| Linux | `晓园-Vault-1.6.1-free.AppImage` | （CI 出）|

> 💡 **Apple Silicon 用户优先下 arm64 版**（原生 arm64 比 x64-on-Rosetta 快 5-15%，电池续航更好）

## ✨ 主要变化

### Skill 模板体系（v1.6 核心）

- **9 个独立 Skill 模板**（`src/main/templates/skills/`）—— 补齐 v1.4 删的 v1.3.1 HTTP 模板遗留，Agent 按触发词激活
- **Skill 模板 ↔ UI 接口对齐**——`conversation-summary` frontmatter 5 字段对齐 MemoryPanel，`lint` 5 类对齐 LintPanel
- **删除 2 个低价值 Skill**（write / list-sessions）—— 跟 ingest 重叠 / 无面板渲染

### 全文搜索升级

- **SearchPanel snippet + 高亮**（SQLite FTS5 `snippet()` 函数原生支持）—— 搜 "ABC" 列表显示 "这是 <mark>ABC</mark> 科技合同, 金额 128 万. 测试 FTS5…"
- **ShortCutGuide 措辞改精确**（`Cmd+F` 搜文档内容 / `Cmd+P` 按名快速切换）

### 开发者工具

- **`scripts/build-free.sh` 修复**—— 双重 `-free` 后缀 + `set -e` 异常时 `trap` 恢复 package.json
- **`scripts/dev-restart.sh`** —— 一键重启 electron-vite dev（解决 vite HMR 对 import 头变更不生效）
- **`scripts/check-memo-import.sh`** —— 扫所有 .tsx 找用 `memo(` 但漏 import 的文件（防止 v1.5 那种 bug 再犯）
- **E2E 测试套件**（`tests/e2e-vault/` + `docs/E2E_TESTING.md`）—— 跨设备手动验证接口对齐，5 步流程 + 6 项验收清单

### 仓库清理

- **删 Pro 仓库残留**（whisper / OCR / 临时测试目录）—— Free 仓库 README 明确"不包含 AI Chat / Bubble / OCR / 语音"
- **`.gitignore` 增强**—— `test-tmp-*/` + `resources/whisper/` + `resources/scripts/`

### 文档

- **README 重写为产品视角**—— 删版本号 / 日期 / version badge / 具体数字，全部导航到 CHANGELOG
- **docs/ 同步到 v1.6.1** —— SKILL_WORKFLOW 重写、ENGINEERING_MAP Free 视角、AGENTS.md 仓库根修复、DEPLOY.md 删 v1.3 AI key
- **CHANGELOG 完整记录 v1.4.1 / v1.5.0 / v1.6.0 / v1.6.1** —— 16 个 commit 摘要

### 安装包优化（dmg -29%）

- **排除已 webpack 处理的包**—— `node_modules/mermaid` 74M 已被 webpack 打成 `vendor-mermaid.js` 5.1M，原包不再装入 asar
- **x64 dmg**：212M → **151M**（-61M）
- **arm64 dmg**：207M → **146M**（-61M）
- 同样机制清理：tesseract.js / pdfjs-dist / xlsx 系列 / echarts / @babel / @antfu 等 13 个包

### Bug 修复

- **10 个 React 组件漏 import `memo`**（v1.5 commit `b453511` 批量加 memo 时漏 import，dev 跑抛 ReferenceError）
  - 第一波 5 个：EditorHeader / FileTreeContextMenu / IndexNav / LintPanel / LogPanel
  - 第二波 5 个：MemoryPanel / OutputPanel / Sidebar / TrashPanel / VersionHistoryPanel
- **`build-free.sh` 双重 `-free` 后缀**—— v1.6.1 package.json 已是 `1.6.1-free`，脚本不应再拼 `-free` 导致 `1.6.1-free-free`
- **`build-free.sh` 异常退出不恢复 package.json**—— 加 `trap EXIT` 防御

## 🔧 升级说明

从 v1.5.0-free 升级：直接覆盖安装（dmg 拖到 /Applications 替换；Windows 卸载重装；Linux 替换 AppImage）。

**Vault 数据兼容**：v1.5.0 创建的 vault 目录结构（含 `_wiki/`、`_briefing/`、`_raw/`、`log.md` 等）无需迁移，v1.6.1 全部兼容。

**Skill 模板变化**（如果用户自改了 `src/main/templates/skills/*.md`）：
- `write.md` 和 `list-sessions.md` **已删除**（v1.6.1 精简）—— 如果用户引用过，请改成手动操作或自定义 Skill
- `conversation-summary.md` frontmatter 字段重命名（`tags` → `topic`，`relatedFiles` → `sources`）

## 📋 已知问题

- **GitHub Actions macos-latest Apple Silicon runner 上可能复现 better-sqlite3 vs Electron binary 冲突**—— 本机验证 build OK（x64 + arm64 dmg 都成功），CI 是全新环境，如果 fail 会单独发 hotfix
- **`test-tmp-*/` 临时目录已被 .gitignore**，但如果之前 commit 过需手动删

## 🧪 如何验证

1. **dmg 验证**：macOS 挂载 dmg 看 `Info.plist` productName = "晓园 Vault (开源版)"
2. **设置面板无 Pro 功能**：启动后看设置面板，**没有**外部 Agent 插件 / 剪贴板浮窗 / AI Chat 浮窗
3. **跨设备 E2E**：参考 `docs/E2E_TESTING.md` 在另一台机器跑 Claude Code / Cursor 验证 7 个 Skill 工作流
